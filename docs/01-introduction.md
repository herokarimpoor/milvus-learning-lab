# Introduction to Milvus

## What Is Milvus?

Milvus is an open-source vector database designed to store, index, and search
high-dimensional vector data efficiently.

Traditional databases are optimized for exact comparisons such as:

```sql
SELECT * FROM users WHERE id = 10;
```

A vector database is designed for similarity questions such as:

- Which face is most similar to this face?
- Which document has the closest meaning to this query?
- Which products are visually similar to this image?
- Which previously stored item is closest to this embedding?

Milvus does not generate embeddings itself. An external model converts an
image, text, audio sample, or another object into a numeric vector. Milvus
stores those vectors and searches for the closest matches.

## What Is a Vector?

A vector is an ordered list of numbers:

```text
[0.12, -0.44, 0.87, 0.09]
```

In real applications, vectors usually contain many more values.

Examples:

- A face-recognition model may produce a 512-dimensional vector.
- A text-embedding model may produce a 768-dimensional vector.
- An image model may produce a 1,024-dimensional vector.

The vector dimension stored in Milvus must exactly match the dimension defined
in the collection schema.

## What Is an Embedding?

An embedding is a vector representation generated by a machine-learning model.

Objects with similar meaning or appearance should have embeddings that are
close to each other in vector space.

For example, a face-recognition pipeline can be represented as:

```text
Face image
    |
    v
Face detection and alignment
    |
    v
Embedding model
    |
    v
512-dimensional vector
    |
    v
Milvus storage and similarity search
```

Milvus stores and searches the final vector. It does not normally store the
original image itself.

## Milvus Concepts

### Collection

A collection is similar to a table in a relational database. It contains a
schema, fields, entities, indexes, and optional partitions.

### Entity

An entity is a stored record. It can contain:

- A primary key
- A vector field
- Scalar metadata such as a name, category, timestamp, or external identifier

### Field

A field defines one part of an entity. Milvus supports scalar fields and vector
fields.

Examples include:

- `INT64`
- `VARCHAR`
- `FLOAT`
- `BOOL`
- `FLOAT_VECTOR`
- `BINARY_VECTOR`

### Schema

A schema defines the fields, data types, primary key, vector dimension, and
other collection rules.

### Index

An index improves vector-search performance. Common indexes include:

- FLAT
- IVF_FLAT
- IVF_SQ8
- HNSW

The best index depends on dataset size, memory, search latency, recall, and
write patterns.

### Metric Type

A metric defines how Milvus compares vectors.

Common metrics include:

- `COSINE` for angular similarity
- `L2` for Euclidean distance
- `IP` for inner product

The metric used during search must be compatible with the metric used to build
the index.

## Milvus Standalone Components

This learning project runs Milvus in standalone mode with three services:

| Component | Responsibility |
|---|---|
| Milvus | Vector operations, collections, indexes, insert, query, and search |
| etcd | Metadata, coordination, and service state |
| MinIO | Persistent object storage for vector data and index files |

These components form one logical vector-database system. A reliable backup
must account for their relationship and consistency.

## Milvus Compared with a Relational Database

| Relational concept | Milvus concept |
|---|---|
| Database table | Collection |
| Row | Entity |
| Column | Field |
| Primary key | Primary field |
| B-tree index | Vector or scalar index |
| Exact query | Scalar query |
| Nearest-neighbor query | Vector search |

Milvus is usually used alongside a relational database rather than as a full
replacement for one.

For example:

- PostgreSQL stores users, permissions, and business records.
- MinIO stores original images.
- Milvus stores image embeddings for similarity search.

## Common Use Cases

Milvus can be used for:

- Face recognition
- Image similarity search
- Semantic text search
- Retrieval-augmented generation
- Recommendation systems
- Audio similarity
- Duplicate detection
- Anomaly detection

## Current Learning Environment

This repository currently uses:

- Milvus 2.3.3
- PyMilvus 2.3.7
- Python 3.10
- etcd 3.5.5
- MinIO
- Docker Compose

These versions are used because they match an existing practical environment.
The concepts in this repository also apply to newer Milvus versions, although
some APIs and configuration options may differ.
